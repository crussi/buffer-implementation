---

Only abstraction improvement I recommend:

## Improvement: buffer should be opaque

Currently:

```c
typedef struct {
    row *rows;
    int numrows;
    int capacity;
} buffer;
```

This exposes internals.

Better:

buffer.h

```c
typedef struct buffer buffer;
```

buffer.c

```c
struct buffer {
    row *rows;
    int numrows;
    int capacity;
};
```

Then expose functions:

```c
int buffer_num_rows(buffer*);
row* buffer_get_row(buffer*, int);
```

This prevents misuse.

Not required now, but recommended long-term.

# 4) Safety issues (VERY IMPORTANT)

You asked the right question. This is where beginners usually introduce crashes.

There are a few real safety issues.

---

## Issue 1 — editorInsertChar has no bounds checking

Dangerous code:

```c
insertChar(&e->buf->rows[row], col, c);
```

If row invalid → undefined behavior.

Fix:

```c
if (!e || row < 0 || row >= e->buf->numrows) return;
```

Same applies to:

editorDeleteChar
editorInsertCR
editorDeleteCR

---

## Issue 2 — editorDeleteChar reads before checking

Dangerous:

```c
char deleted = e->buf->rows[row].line[col];
```

If col invalid → crash.

Fix:

```c
if (col < 0 || col >= e->buf->rows[row].length) return;
```

---

## Issue 3 — buffer.c realloc failure handling is incomplete

Example:

```c
row* tmp = realloc(buf->rows, newCap * sizeof(row));
if (!tmp) {
    free(line);
    free(buf);
    return NULL;
}
```

You forgot to free existing rows inside buf.

This causes memory leak.

Should be:

```c
for (int i = 0; i < buf->numrows; i++)
    free(buf->rows[i].line);
free(buf->rows);
free(buf);
```

---

## Issue 4 — ActionStack ownership contract is fragile

push_action copies Action struct shallowly:

```c
stack->actions[stack->size++] = action;
```

This assumes:

action.text must be heap allocated OR NULL

This must be documented clearly.

Otherwise double-free or invalid free will occur.

---

## Issue 5 — insertChar allows negative at values silently

You clamp it:

```c
if (at <= 0) at = 0;
```

This is acceptable design.

Not unsafe.
